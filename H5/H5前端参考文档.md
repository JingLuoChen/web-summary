# H5前端参考文档

## 注意事项

1. **对设计稿的要求**


```

一般情况下，H5页面设计稿做成750x1334的寸尺（以iphone6尺寸为基础）并且在812px高度处增加一条安全线，重要内容在此线之上，既保证了在移动设备上显示清晰，也保证了素材的最小尺寸

iphone6尺寸：375 x 667 px (750 x 1334 px@2x)
顶部通栏+微信导航条高度：20px + 44px = 64px (128px@2x)
H5页面高度：667px - 64px = 603px（1206px@2x）

```

2. **全局边距**

``` 

	全局边距是指页面内容到屏幕边距的距离，整个应用的界面都应该以此来进行规范，以达到页面整体视觉效果的统一
	通常左右边距最小为20px，这样的距离可以展示更多的内容，不建议比20还小，否则就会是界面内容过于拥挤，给用户的浏览带来视觉负担，30px是非常舒服的距离，是绝大多数应用的首选边距

```

3. **对素材图片准备、处理和使用**

``` 
	切图准备：
	
	单页切图准备：单页切图推荐内容全部单独切出来，保留底部背景图片，使展示内容尽量展示在单页屏幕上，避免因为裁剪导致部分内容因在背景图片中而被裁减掉
	
	滚屏切图准备：可滚屏页面图片切图可视情况而定，部分情况可保留相关内容在背景图片上【保留文字的前提是不设置不同屏幕分辨率下字体大小一样，因不同分辨率下图片上的文字会跟随放大，字体大小因此没有限制】

	使用前：
	
	使用前请经过压缩处理，图片压缩可以使用tinypng【地址：https://tinypng.com/】
	素材背景图片大小为500kb以下，其余素材图片100px以下,尽量使用jpg文件

	H5中使用图片推荐上传到cdn上，缓解因大量访问造成服务器压力
	图片上传cdn地址：
	  https://tech-docs.gotokeep.com/keepcli/#/zh-cn/cdn/ 或者 
    https://cms.pre.gotokeep.com/#/resource/upload 
                              
  使用时：
  
  注意：图片使用避免因设置宽高而导致压缩变形的情况，以及因适配导致图片压缩变形的情况，
  解决：可以对图片设置宽度（高度），进行等比缩放（或者设置成背景图片使用background-position: center center ）        
  
  注意：背景图片使用background-size:cover来实现，cover主要用来把背景图像扩展到足够大，使其完全覆盖背景区域。【不使用img原因：background会等到结构加载完成才开始加载，而img是网页结构的一部分会在加载结构的过程中加载，img会导致后面的内容都不会显示影响网页内容】
	
```

4. **界面文字设计规范**

``` 
	
	字号选择：建议最小不超过18px,其他字号双数，同种类别下用一样字号
	
	注意：手机上最小的字体大小是8px,pc端的最小的字体大小是12px
	
	如需保证不同屏幕看到的字号时相同的话，可以设置不同dpr下的字体大小
	
	代码
    div {
      width: 1rem; 
      height: 0.4rem;
      font-size: 12px; // 默认写上dpr为1的fontSize
    }
    [data-dpr="2"] div {
       font-size: 24px;
    }
    [data-dpr="3"] div {
        font-size: 36px;
    }
	
  
```

5. **页面布局**

``` 

1. 通过使用视口单位实现布局（不兼容opera浏览器）

	视口单位
		vw：1vw等于视口宽度的1%
		vh: 1vh等于视口高度的1%
		vmin: 选取vw和vh中最小的那个
		vmax:选取vw和vh中最大的那个
		
	兼容性：在移动端ios8以上以及android4.4以上获得支持，并且在微信x5内核中也得到完美的全面支持
	
	资料网址：https://aotu.io/notes/2017/04/28/2017-4-28-CSS-viewport-units/index.html
	
	区别：视口单位区别于%单位，视口单位是依赖于视口的尺寸，根据视口尺寸的百分比来定义，而%单位则是依赖于元素的祖先元素

2.通过使用百分比实现布局
	
	百分比的具体分析
		子元素的宽高的百分比 --- 是相对于子元素的直接父元素
		子元素的top和bottom的百分比 --- 是相对于直接非static定位的父元素的高度
		子元素的left和right的百分比 --- 是相对于直接非static定位的父元素的宽度
		子元素的padding和margin的百分比 --- 不论处于垂直方向或者水平方向，都是相对于直接父元素的宽度
		子元素的border-radius的百分比 --- 则是相对于自身的宽度
		
	缺点：计算困难，单位需要换算成百分比单位、相对父元素的属性并不是唯一的
		
3.通过使用rem实现布局
	
	rem是相对于浏览器的根元素的font-size来决定大小的，根元素的font-size相当于提供了一个基准，当页面的size发生变化时，只需要改变font-size的值，那么以rem为固定单位的元素的大小也会发生响应的变化
	
	【页面根元素的计算】
	
   var fontSize = screen_width / psdWidth * 100;
  
   乘100的原因： 手机一般浏览器的最小字体是8px 防止出现0.3px等情况 被转为8px 造成误差

	
注意事项：
 
		1）、进行页面规划的时候，不应该把重要的内容放在太偏上或者太偏下，否则布局会紊乱
		
		2）、在H5页面中的定位推荐使用top,尽量不要top以及bottom一起使用，目前iphonex的屏幕比较长，如果按照屏幕底部布局以及顶部布局结合来，会出现你意想不到的效果
		
		注意：因H5活动一般变化比较频繁，素材图片等会出现频繁更新的情况，前期调整好布局，等最终替换素材（开发人员可推动设计尽早提供素材）


```
